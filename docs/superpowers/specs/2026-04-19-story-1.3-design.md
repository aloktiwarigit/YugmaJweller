## Part A вАФ Database migration (0007)

File: `packages/db/src/migrations/0007_staff_invite_columns.sql`

```sql
-- 0007_staff_invite_columns.sql
ALTER TABLE shop_users
  ADD COLUMN invited_by_user_id UUID REFERENCES shop_users(id),
  ADD COLUMN invited_at TIMESTAMPTZ;
```

- Both columns nullable вАФ OWNER rows created at provisioning have no inviter.
- No RLS change needed; existing `shop_id`-scoped policy covers new columns.
- `invited_at` is set explicitly in the INSERT (not a column default) so the timestamp reflects the actual invite action.

Drizzle schema update: `packages/db/src/schema/shop-users.ts` gets both columns added to `shopUsers`.

## Part A вАФ Invite endpoint

### DTO

`apps/api/src/modules/auth/dto/invite-staff.dto.ts`

```ts
export class InviteStaffDto {
  phone: string;        // E.164 format, e.g. "+919876543210"
  role: 'shop_manager' | 'shop_staff';
}
```

Zod schema shared with frontend: `packages/schemas/src/auth.ts` gets `inviteStaffSchema`.

### Controller

`POST /api/v1/auth/invite` on `AuthController`:
- `@TenantWalkerRoute()` вАФ sets tenant context from JWT claims
- `@Roles('shop_admin')` вАФ guard blocks non-owners with `403 { code: 'auth.insufficient_role' }`
- Returns `201 { id, phone, role, status: 'INVITED', invited_at }`

### Service вАФ `AuthService.invite()`

```
args: { actorUserId, shopId, phone (E.164), role, tenant }

1. Validate phone E.164 format (regex)
2. AuthRepository.findByPhone(shopId, phone)
   вЖТ existing INVITED or ACTIVE row вЖТ throw 409 { code: 'auth.staff_already_exists' }
3. AuthRepository.insertInvited({ shopId, phone, role, invitedByUserId, invitedAt: NOW() })
4. sms.send(phone, inviteMessage)   вЖР StubSmsAdapter logs only
5. auditLog: STAFF_INVITED, before: null, after: { phone, role, status: 'INVITED' }
6. Return inserted row fields
```

### Repository вАФ `AuthRepository`

New methods:
- `findByPhoneInShop(shopId, phone)` вАФ SELECT inside `withTenantTx`; returns status or null
- `insertInvited(args)` вАФ INSERT inside `withTenantTx`; returns new row

### Staff list endpoint

`GET /api/v1/auth/staff`:
- `@TenantWalkerRoute()` + `@Roles('shop_admin', 'shop_manager')`
- Returns array of `{ id, phone, display_name, role, status, invited_at }` for the caller's shop

### `(tabs)/_layout.tsx` changes

Reads `useAuthStore(s => s.user?.role)`. Uses `href: null` (expo-router native) to hide tabs without breaking file-based routing.

```tsx
const role = useAuthStore(s => s.user?.role);
const isManagerOrOwner = role === 'shop_manager' || role === 'shop_admin';

<Tabs screenOptions={{ headerShown: false }}>
  <Tabs.Screen name="index"   options={{ title: t('dashboard.tabs.dashboard') }} />
  <Tabs.Screen name="billing" options={{ title: t('dashboard.tabs.billing'),
                                          href: null }} />  {/* Epic 5 вАФ always stub */}
  <Tabs.Screen name="reports" options={{ title: t('dashboard.tabs.reports'),
                                          href: isManagerOrOwner ? undefined : null }} />
</Tabs>
```

Note: billing tab uses `href: null` for all roles in story 1.3 (Epic 5 stub). The file exists so expo-router doesn't throw. When Epic 5 ships, remove `href: null`.

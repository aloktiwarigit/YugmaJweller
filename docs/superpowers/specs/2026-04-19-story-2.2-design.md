## API Design

### `GET /api/v1/settings/making-charges`

- Auth: OWNER (`shop_admin`) or MANAGER (`shop_manager`)
- Cache-first: `SettingsCache.getMakingCharges()` → DB miss → inject defaults if null
- Returns: `MakingChargesResponseDto` (array of `MakingChargeConfig` + `etag`)
- ETag: `sha256(JSON.stringify(configs)).slice(0,16)` — same pattern as profile

### `PATCH /api/v1/settings/making-charges`

- Auth: OWNER only (`shop_admin`)
- Body: `PatchMakingChargesDto` — array of `MakingChargeConfig` (partial update — only supplied categories are written; others retain existing/default values)
- Zod validation (enforced in service, not controller):
  - `value` must be numeric string, parseable to DECIMAL, `> 0`
  - `type === 'percent'` → `value ≤ 100`
  - `type === 'fixed_per_gram'` → no upper bound
- Upserts `shop_settings.making_charges_json` as full array (merge supplied with existing defaults)
- On save: invalidate `making_charges` cache key, emit audit `SETTINGS_MAKING_CHARGES_UPDATED { before, after }`
- Returns: full `MakingChargesResponseDto` (all 6 categories, merged) + ETag

## Schema (packages/shared)

### `making-charges.schema.ts` (new file)

```typescript
export const ProductCategory = z.enum([
  'RINGS', 'CHAINS', 'BANGLES', 'BRIDAL', 'SILVER', 'WHOLESALE'
]);

export const MakingChargeConfigSchema = z.object({
  category: ProductCategory,
  type: z.enum(['percent', 'fixed_per_gram']),
  value: z.string()
    .refine(s => /^\d+(\.\d+)?$/.test(s) && parseFloat(s) > 0, 'VALUE_POSITIVE_REQUIRED')
    // percent ≤ 100 enforced in PatchMakingChargesSchema superRefine
});

export const PatchMakingChargesSchema = z.array(MakingChargeConfigSchema)
  .superRefine((items, ctx) => {
    for (const [i, item] of items.entries()) {
      if (item.type === 'percent' && parseFloat(item.value) > 100) {
        ctx.addIssue({ code: z.ZodIssueCode.custom, message: 'PERCENT_MAX_100', path: [i, 'value'] });
      }
    }
  });
```

**Exported types:** `ProductCategory`, `MakingChargeConfig`, `PatchMakingChargesDto`, `MakingChargesRow`

## Tests Required

| Test | Type | What it verifies |
|---|---|---|
| `updateMakingCharges` — Zod rejects float value (`"1.5e2"` not parseable) | Unit | No float coercion |
| `updateMakingCharges` — rejects negative value | Unit | `> 0` guard |
| `updateMakingCharges` — rejects percent > 100 | Unit | `PERCENT_MAX_100` |
| `updateMakingCharges` — stores value as string, not number | Unit | `JSON.parse(stored)` is string |
| `updateMakingCharges` — cache invalidated after upsert | Unit | `cache.invalidateMakingCharges` called |
| `updateMakingCharges` — audit emitted with before/after | Unit | `auditLog` called with `SETTINGS_MAKING_CHARGES_UPDATED` |
| PATCH → DB → GET round-trip, value precision preserved | Integration | `"10.00"` in → `"10.00"` out |
| Tenant isolation: tenant A JWT cannot update tenant B row | Integration | 403 or DB RLS rejects |

# Story 2.3 — Wastage Configuration Design Spec

**Date:** 2026-04-19
**Class:** B (Codex + CI + curl smoke test)
**PRD FRs:** FR-51 (shopkeeper configures wastage %), FR-52 (per-category defaults), FR-53 (making charges / wastage DECIMAL precision)

---

## What This Builds

`GET + PATCH /api/v1/settings/wastage` and a mobile `WastageRow` component. Pure pattern repeat of Story 2.2 (making charges) with three simplifications: percent-only (no type toggle), single-item PATCH body, stricter ≤30% domain cap.

---

## Architecture

Three-layer pattern identical to making-charges:

```
Controller (auth guard, ETag)
  └─ Service (Zod parse, >30% guard, cache-aside, audit)
       └─ Repository (withTenantTx, SELECT FOR UPDATE, UPSERT)
            └─ PostgreSQL shop_settings.wastage_json (JSONB, no migration)
```

Cache layer: `SettingsCache` gains per-key wastage methods. Key `shop:{shopId}:settings:wastage`, TTL 60s, independent of profile and making-charges keys.

## Shared Schema

**New file:** `packages/shared/src/schemas/wastage.schema.ts`

```typescript
import { z } from 'zod';
import { ProductCategory } from './making-charges.schema';

export { ProductCategory };  // re-export so consumers import from one place

const positiveDecimalString = z
  .string()
  .regex(/^\d+(\.\d+)?$/, 'VALUE_FORMAT_INVALID')
  .refine((s) => parseFloat(s) > 0, 'VALUE_POSITIVE_REQUIRED');

export const WastageConfigSchema = z.object({
  category: ProductCategory,
  percent:  positiveDecimalString,
});
export type WastageConfig = z.infer<typeof WastageConfigSchema>;

export const PatchWastageSchema = WastageConfigSchema;
export type PatchWastageDto = z.infer<typeof PatchWastageSchema>;

export const WastageArraySchema = z.array(WastageConfigSchema);

export const WASTAGE_DEFAULTS: WastageConfig[] = [
  { category: 'RINGS',     percent: '2.00' },
  { category: 'CHAINS',    percent: '1.50' },
  { category: 'BANGLES',   percent: '1.50' },
  { category: 'BRIDAL',    percent: '3.00' },
  { category: 'SILVER',    percent: '1.00' },
  { category: 'WHOLESALE', percent: '1.00' },
];
```

Exports added to `packages/shared/src/index.ts`.

## Cache Extension

`packages/tenant-config/src/settings-cache.ts` — three new methods alongside existing making-charges methods:

| Method | Description |
|--------|-------------|
| `getWastage()` | `GET` cache key, Zod-validate, return null on miss/corrupt |
| `setWastage(data)` | `SET EX 60` |
| `invalidateWastage()` | `DEL` the wastage key only |

**Cache key:** `shop:{shopId}:settings:wastage`

Invalidation is per-key — a wastage write does not bust the making-charges or profile cache.

## API Layer

### Repository (`settings.repository.ts`)

Two new methods:

**`getWastage(): Promise<Record<string, string> | null>`**
```sql
SELECT wastage_json FROM shop_settings WHERE shop_id = $1
```
Returns the raw JSONB map (e.g. `{ "BRIDAL": "2.50" }`) or null if no row / null column.

**`upsertWastage(category, percent): Promise<UpdateWastageResult>`**
```sql
INSERT INTO shop_settings (shop_id) VALUES ($1) ON CONFLICT (shop_id) DO NOTHING;
SELECT wastage_json FROM shop_settings WHERE shop_id = $1 FOR UPDATE;
-- merge: existingMap[category] = percent
INSERT INTO shop_settings (shop_id, wastage_json)
VALUES ($1, $2::jsonb)
ON CONFLICT (shop_id)
DO UPDATE SET wastage_json = $2::jsonb, updated_at = now()
RETURNING wastage_json;
```

Returns `{ before: WastageConfig[] | null, after: WastageConfig[] }` where both sides are fully-expanded 6-category arrays (defaults merged).

### Service (`settings.service.ts`)

**`getWastage(): Promise<WastageConfig[]>`**
- Cache-aside: hit → return. Miss → repo → merge over `WASTAGE_DEFAULTS` → set cache → return.
- Merge: `WASTAGE_DEFAULTS.map(d => storedMap?.get(d.category) ?? d)` where stored map is `new Map(Object.entries(storedRaw))`.

**`updateWastage(dto: PatchWastageDto): Promise<WastageConfig[]>`**
1. `PatchWastageSchema.safeParse(dto)` — 400 on format failure
2. `if (parseFloat(parsed.data.percent) > 30)` → throw `UnprocessableEntityException({ code: 'settings.wastage_high', message: 'घटत 30% से ज़्यादा नहीं होनी चाहिए' })`
3. `repo.upsertWastage(category, percent)`
4. `cache.invalidateWastage()`
5. `auditLog(SETTINGS_WASTAGE_UPDATED, { before, after })` — fire-and-forget
6. Return `after`

### Controller (`settings.controller.ts`)

```
GET  /api/v1/settings/wastage  — OWNER + MANAGER
PATCH /api/v1/settings/wastage — OWNER only
```

Same auth guard pattern as making-charges. ETag computed as `sha256(JSON.stringify(configs)).slice(0,16)`.

### DTO (`settings.dto.ts`)

```typescript
export interface WastageResponseDto {

```

### Audit (`audit-actions.ts`)

Add `SETTINGS_WASTAGE_UPDATED = 'SETTINGS_WASTAGE_UPDATED'` to `AuditAction` enum.

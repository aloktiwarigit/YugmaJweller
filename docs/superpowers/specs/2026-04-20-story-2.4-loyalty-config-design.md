# Story 2.4 — Loyalty Program Configuration Design Spec

**Date:** 2026-04-20  
**Class:** B (Codex + CI + curl smoke test)  
**PRD FRs:** FR-60 (shopkeeper configures loyalty tiers), FR-61 (earn/redemption rates), FR-62 (tier badge display)

---

## What This Builds

`GET + PATCH /api/v1/settings/loyalty` and a mobile `LoyaltyTierForm` component. Same three-layer pattern as Stories 2.2 (making charges) and 2.3 (wastage). Key differences: paise arithmetic for tier thresholds, discriminated union PATCH body for two sub-operations (tier update vs rate update), and tier ordering invariant enforced at service layer.

---

## Decisions Locked

| # | Question | Decision |
|---|----------|----------|
| 1 | Paise storage format in JSONB | **Integer number** (e.g. `15000000`). PostgreSQL JSONB is exact for integers in JS safe integer range. Validate as `Number.isInteger(v)` on every read — never cast to float. |
| 2 | Single endpoint vs two | **Single discriminated-union PATCH** (`{ type: "tier" \| "rate" }`). One ETag, one cache invalidation path, fewer route handlers. |

---

## JSONB Structure

Stored in `shop_settings.loyalty_json` (column exists from migration 0006, no migration needed):

```json
{
  "tiers": [
    { "name": "Silver",  "thresholdPaise": 5000000,  "badgeColor": "#C0C0C0" },
    { "name": "Gold",    "thresholdPaise": 15000000,  "badgeColor": "#FFD700" },
    { "name": "Diamond", "thresholdPaise": 50000000,  "badgeColor": "#B9F2FF" }
  ],
  "earnRatePercentage": "1.00",
  "redemptionRatePercentage": "1.00"
}
```

- `thresholdPaise` is a native JSON integer. Validate as `Number.isInteger(v) && v >= 0 && v <= 1_000_000_00` on read.
- Rate percentages are DECIMAL strings — never parsed to float for storage.
- All three tier slots always present (sparse representation not permitted).

---

## Architecture

```
Controller (auth guard, ETag)
  └─ Service (Zod parse, paise conversion, tier-ordering guard, cache-aside, audit)
       └─ Repository (withTenantTx, SELECT FOR UPDATE, UPSERT)
            └─ PostgreSQL shop_settings.loyalty_json
```

Cache layer: `SettingsCache` gains three new methods. Key `shop:{shopId}:settings:loyalty`, TTL 60 s, independent of profile / making-charges / wastage keys.

## Shared Schema

**New file:** `packages/shared/src/schemas/loyalty-config.schema.ts`

```typescript
import { z } from 'zod';

const HEX_COLOR_REGEX = /^#[0-9A-Fa-f]{6}$/;

const positiveDecimalString = z
  .string()
  .regex(/^\d+(\.\d+)?$/, 'VALUE_FORMAT_INVALID')
  .refine((s) => parseFloat(s) > 0 && parseFloat(s) <= 100, 'RATE_OUT_OF_RANGE');

export const LoyaltyTierSchema = z.object({
  name:           z.string().min(1, 'TIER_NAME_REQUIRED').max(20, 'TIER_NAME_TOO_LONG'),
  thresholdPaise: z.number().int('THRESHOLD_MUST_BE_INTEGER').min(0, 'THRESHOLD_MIN').max(1_000_000_00, 'THRESHOLD_MAX'),
  badgeColor:     z.string().regex(HEX_COLOR_REGEX, 'BADGE_COLOR_INVALID'),
});
export type LoyaltyTier = z.infer<typeof LoyaltyTierSchema>;

export const LoyaltyConfigSchema = z.object({
  tiers:                    z.tuple([LoyaltyTierSchema, LoyaltyTierSchema, LoyaltyTierSchema]),
  earnRatePercentage:       positiveDecimalString,
  redemptionRatePercentage: positiveDecimalString,
});
export type LoyaltyConfig = z.infer<typeof LoyaltyConfigSchema>;

// Input schemas for PATCH body
const thresholdRupeesString = z
  .string()
  .regex(/^\d+(\.\d{1,2})?$/, 'VALUE_FORMAT_INVALID')
  .refine((s) => parseFloat(s) >= 0 && parseFloat(s) <= 100_000_00, 'THRESHOLD_OUT_OF_RANGE');

export const PatchLoyaltyTierSchema = z.object({
  type:            z.literal('tier'),
  index:           z.union([z.literal(0), z.literal(1), z.literal(2)]),
  name:            z.string().min(1, 'TIER_NAME_REQUIRED').max(20, 'TIER_NAME_TOO_LONG'),
  thresholdRupees: thresholdRupeesString,
  badgeColor:      z.string().regex(HEX_COLOR_REGEX, 'BADGE_COLOR_INVALID'),
});
export type PatchLoyaltyTierDto = z.infer<typeof PatchLoyaltyTierSchema>;

export const PatchLoyaltyRateSchema = z.object({
  type:                     z.literal('rate'),
  earnRatePercentage:       positiveDecimalString,
  redemptionRatePercentage: positiveDecimalString,
});
export type PatchLoyaltyRateDto = z.infer<typeof PatchLoyaltyRateSchema>;

export const PatchLoyaltySchema = z.discriminatedUnion('type', [
  PatchLoyaltyTierSchema,
  PatchLoyaltyRateSchema,
]);
export type PatchLoyaltyDto = z.infer<typeof PatchLoyaltySchema>;

export const LOYALTY_DEFAULTS: LoyaltyConfig = {
  tiers: [
    { name: 'Silver',  thresholdPaise: 5_000_000,  badgeColor: '#C0C0C0' },
    { name: 'Gold',    thresholdPaise: 15_000_000,  badgeColor: '#FFD700' },
    { name: 'Diamond', thresholdPaise: 50_000_000,  badgeColor: '#B9F2FF' },
  ],
  earnRatePercentage:       '1.00',
  redemptionRatePercentage: '1.00',
};
```

## Paise Conversion

Centralised helper (inline in service, not a package):

```typescript
function rupeesToPaise(rupeesStr: string): number {
  const paise = Math.round(parseFloat(rupeesStr) * 100);
  if (!Number.isInteger(paise) || paise < 0 || paise > 1_000_000_00) {
    throw new UnprocessableEntityException({ code: 'settings.threshold_out_of_range' });
  }
  return paise;
}
```

Inverse for display: `(paise / 100).toFixed(2)` — used in response DTO only, never stored.

---

## Tier Ordering Invariant

After any tier update, validate before saving:

```typescript
if (
  updated[0].thresholdPaise >= updated[1].thresholdPaise ||
  updated[1].thresholdPaise >= updated[2].thresholdPaise
) {
  throw new UnprocessableEntityException({
    code: 'settings.tier_order_invalid',
    message: 'Tier thresholds बढ़ते क्रम में होने चाहिए',
  });
}
```

This 422 fires before the DB write. Error is surfaced inline in the mobile form before blur-save is attempted.

## Cache Extension

`packages/tenant-config/src/settings-cache.ts` — three new methods:

| Method | Description |
|--------|-------------|
| `getLoyalty()` | GET cache key, Zod-validate (`LoyaltyConfigSchema`), return null on miss/corrupt |
| `setLoyalty(data)` | SET EX 60 |
| `invalidateLoyalty()` | DEL the loyalty key only |

**Cache key:** `shop:{shopId}:settings:loyalty`

Invalidation is key-scoped — loyalty write does not bust profile, making-charges, or wastage caches.

## API Layer

### Repository additions (`settings.repository.ts`)

**`getLoyalty(): Promise<LoyaltyConfig | null>`**
```sql
SELECT loyalty_json FROM shop_settings WHERE shop_id = $1
```
Returns parsed config (after Zod validation) or null. Null means use defaults.

**`upsertLoyalty(config: LoyaltyConfig): Promise<UpdateLoyaltyResult>`**
```sql
-- Ensure row exists
INSERT INTO shop_settings (shop_id) VALUES ($1) ON CONFLICT (shop_id) DO NOTHING;
-- Read-for-update (prevents concurrent writes)
SELECT loyalty_json FROM shop_settings WHERE shop_id = $1 FOR UPDATE;
-- Upsert full config
INSERT INTO shop_settings (shop_id, loyalty_json)
VALUES ($1, $2::jsonb)
ON CONFLICT (shop_id)
DO UPDATE SET loyalty_json = $2::jsonb, updated_at = now()
RETURNING loyalty_json;
```
Returns `{ before: LoyaltyConfig | null, after: LoyaltyConfig }`.

### Service additions (`settings.service.ts`)

**`getLoyalty(): Promise<LoyaltyConfig>`**
- Cache-aside: hit → parse → return. Miss → repo → merge over `LOYALTY_DEFAULTS` → set cache → return.
- Merge strategy: stored config overrides defaults field-by-field; if stored is null, return `LOYALTY_DEFAULTS`.

**`updateLoyalty(dto: PatchLoyaltyDto): Promise<LoyaltyConfig>`**
1. `PatchLoyaltySchema.safeParse(dto)` — 400 on format failure
2. Load current config (cache-aside, same as getLoyalty)
3. Apply patch:
   - `type === "tier"`: convert `thresholdRupees` → paise; set `current.tiers[index]`; validate tier ordering (422)
   - `type === "rate"`: set `earnRatePercentage`, `redemptionRatePercentage`
4. `repo.upsertLoyalty(updated)`
5. `cache.invalidateLoyalty()`
6. `auditLog(SETTINGS_LOYALTY_UPDATED, { before, after })` — fire-and-forget
7. Return `after`

### Controller additions (`settings.controller.ts`)

```
GET  /api/v1/settings/loyalty  — shop_admin + shop_manager
PATCH /api/v1/settings/loyalty — shop_admin only
```

ETag: `sha256(JSON.stringify(config)).slice(0, 16)`.

Auth guard pattern identical to `/profile` and `/wastage`.

## Mobile

### `LoyaltyTierForm.tsx`

**Location:** `apps/shopkeeper/src/features/settings/components/LoyaltyTierForm.tsx`

Shape:
```typescript
interface Props {
  config: LoyaltyConfig;
  onChangeTier: (index: 0 | 1 | 2, tier: Partial<LoyaltyTier>) => void;
  onChangeRate: (rate: Pick<LoyaltyConfig, 'earnRatePercentage' | 'redemptionRatePercentage'>) => void;
  onBlurSave: (dto: PatchLoyaltyDto) => Promise<void>;
  saving: boolean;
  error: string | null;
}
```

Layout (scroll view):
1. **Section: Tier thresholds** (3 rows)
   - Each row: badge color circle (16dp) + Hindi name input + threshold input (₹, Indian number format hint text "₹0") + inline ordering warning if violated
   - Save on blur of name or threshold field
2. **Section: Earn rate** — "हर ₹100 खर्च पर X points" with X as TextInput
3. **Section: Redemption rate** — "100 points = ₹Y छूट" with Y as TextInput
4. **Ordering error banner** — shown inline above tier rows if `tier_order_invalid` is returned; does not block viewing but highlights affected field

Touch targets: ≥ 48 dp. Font size: 16pt body, 14pt secondary labels. High contrast.

## Tests (TDD — Red → Green → Refactor)

### Unit (`settings.service.test.ts`)

| Test | Assertion |
|------|-----------|
| Paise conversion — ₹50,000 → 5000000 | `rupeesToPaise("50000") === 5000000` |
| Paise conversion — ₹1,23,456.78 → 12345678 | no float drift; result is integer |
| Tier ordering — out-of-order → 422 | `code: "settings.tier_order_invalid"` |
| Tier ordering — equal thresholds → 422 | strict inequality enforced |
| DECIMAL rate strings — stored as string not float | `earnRatePercentage === "1.00"` not `1` |
| thresholdPaise stored as integer in JSONB | `Number.isInteger(tier.thresholdPaise)` |
| Cache invalidation on update | `invalidateLoyalty()` called, not `invalidate()` |
| Audit log on update | `SETTINGS_LOYALTY_UPDATED` with before/after |
| getLoyalty — cache hit | repo not called |
| getLoyalty — null DB row → LOYALTY_DEFAULTS | defaults returned |

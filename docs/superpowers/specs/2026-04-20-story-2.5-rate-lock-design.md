**Date:** 2026-04-20  
**Class:** B — Codex + CI + curl smoke test

**Complexity:** XS — simplest Epic 2 story; single integer column, no JSONB, no category map

---

## What we are building

A shopkeeper-configurable rate-lock window (1–30 days, default 7). When a customer views a product the shopkeeper can guarantee that quoted price is honoured for this many days. Stored in `shop_settings.rate_lock_days` (column already exists from migration 0006).

PRD traceability: FR-48 (rate-lock duration), NFR-12 (shopkeeper self-service config).

## Settings cache — `packages/tenant-config`

Three new methods following the exact pattern as `getWastage / setWastage / invalidateWastage`. The cached value is a plain `number` — no Zod array schema required; validate with `typeof parsed === 'number' && Number.isInteger(parsed)`.

```ts
async getRateLock(): Promise<number | null>
async setRateLock(days: number): Promise<void>
async invalidateRateLock(): Promise<void>

private rateLockKey(): string {
  return `shop:${tenantContext.requireCurrent().shopId}:settings:rate_lock`;
}
```

Cache errors on `getRateLock` are swallowed as cache misses (delete corrupt key). `setRateLock` / `invalidateRateLock` propagate Redis errors.

TTL: same `DEFAULT_TTL_SEC = 60` as other keys.

## Settings repository

### `getRateLockDays(): Promise<number | null>`

```sql
SELECT rate_lock_days FROM shop_settings WHERE shop_id = $1
```

Returns `null` if no row exists or column is NULL.

### `updateRateLockDays(days: number): Promise<{ before: number | null; after: number }>`

```sql
INSERT INTO shop_settings (shop_id, rate_lock_days, updated_at)
VALUES ($1, $2, now())
ON CONFLICT (shop_id) DO UPDATE
  SET rate_lock_days = EXCLUDED.rate_lock_days,
      updated_at     = now()
RETURNING rate_lock_days
```

Reads `before` with a separate `SELECT` inside `withTenantTx` before the upsert. Returns `{ before, after }`.

Both methods execute inside `withTenantTx` — tenant RLS is enforced by the transaction context.

## Settings service

```ts
async getRateLock(): Promise<number> {
  const hit = await this.cache.getRateLock();
  if (hit !== null) return hit;
  const stored = await this.repo.getRateLockDays();
  const days = stored ?? RATE_LOCK_DEFAULT_DAYS;
  await this.cache.setRateLock(days);
  return days;
}

async updateRateLock(dto: PatchRateLockDto): Promise<number> {
  const parsed = PatchRateLockSchema.safeParse(dto);
  if (!parsed.success) {
    const errors = parsed.error.issues.map((i) => ({ field: i.path.join('.'), code: i.message }));
    throw new UnprocessableEntityException({ code: 'validation.failed', errors });
  }

  const { before, after } = await this.repo.updateRateLockDays(parsed.data.rateLockDays);
  await this.cache.invalidateRateLock();
  void this.auditRateLockUpdate(before, after).catch(() => undefined);
  return after;
}
```

`UnprocessableEntityException` (422) chosen over `BadRequestException` (400) to match the task spec and the pattern established in `updateWastage`.

### Clamp + shake

When user taps `+` at 30 or `−` at 1:
1. Clamp value (no change)
2. Trigger `Animated.sequence([...shake oscillations])` — 3 cycles of ±4px horizontal shift, 60ms each, no `useNativeDriver` since we're animating `translateX`

### Debounce implementation

```ts
const debounceRef = useRef<ReturnType<typeof setTimeout> | null>(null);

function handleStep(delta: number): void {
  const next = Math.min(30, Math.max(1, localDays + delta));
  if (next === localDays) { triggerShake(); return; }
  setLocalDays(next);
  if (debounceRef.current) clearTimeout(debounceRef.current);
  debounceRef.current = setTimeout(() => void save(next), 1000);
}

// cleanup on unmount
useEffect(() => () => { if (debounceRef.current) clearTimeout(debounceRef.current); }, []);
```

### Save flow

```ts
async function save(days: number): Promise<void> {
  setSaving(true);
  try {
    await onSave(days);
    await Haptics.impactAsync(ImpactFeedbackStyle.Medium);
    setShowSuccess(true);
    setTimeout(() => setShowSuccess(false), 2000);
    setError(null);
  } catch (err) {
    setError(t('settings.rate_lock.errors.RANGE_INVALID'));
  } finally {
    setSaving(false);
  }
}
```
